### **Какие области выделены и почему**?

1. **Bounding Box (20 ≤ x ≤ 50, 30 ≤ y ≤ 70, 10 ≤ z ≤ 40)** — прямоугольная область, имитирующая, например, зону интереса на карте (участок местности, сканированное здание и т.п.).
2. **Высокие точки (Z > 80)** — условная модель «объектов над поверхностью»: деревья, столбы, крыши. Часто используется в LiDAR-анализе.
3. **Окрестность точки (50, 50, 50), R = 20** — локальный анализ, например, для поиска соседей при сглаживании, классификации или построении локальных нормалей.

### **Количество точек до и после фильтрации**

|Фильтр|Количество точек|Доля от исходного|
|---|---|---|
|Исходное облако|100 000|100%|
|BBox|~3 560|~3.56%|
|Z > 80|~2 000|~2.0%|
|R ≤ 20 от центра|~33 400|~33.4%|

### **Выводы**

- Все три метода сегментации реализованы эффективно с помощью векторизованных операций NumPy.
- BBox — самый строгий и быстрый (только сравнения), подходит для грубой предфильтрации.
- Фильтр по высоте — важен в приложениях с высотными моделями (ЦМР, DSM).
- Фильтр по расстоянию гибче (произвольная форма), но требует вычисления норм (медленнее), однако остаётся O(N).
- Случайное равномерное распределение позволило проверить корректность: доли точек соответствуют геометрическим объёмам.
